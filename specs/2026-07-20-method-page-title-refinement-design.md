# 方法页标题微调设计

- 日期：2026-07-20
- 状态：待用户复核
- 页面：`/methods/`

## 目标

减少方法页首屏的解释性文字，让标题同时承接“从分析到决策”和“从技术到应用”两条主线。

## 修改

1. 顶部导航继续使用“方法”，保持与“项目、公开作品、关于”的简短同级关系。
2. 方法页眉题继续使用“研究、项目与公开作品中的做法”。
3. 方法页 H1 从“方法”改为“判断与实现”。
4. 删除 H1 下方整段说明：“这里记录我在研究、写作和系统建设中反复使用的做法。我会根据新材料和项目结果，调整这些做法。”
5. 标题之后直接进入现有方法索引，不修改七个入口、两条主线或页面布局。

## 验证

- 更新方法页内容测试，不改站点导航名称。
- 运行完整测试、静态构建与链接检查。
- 检查桌面端和 390px 手机端的标题换行、留白与横向溢出。
- 刷新已提交的静态站点产物。

## 不包含

- 不修改首页的“方法”栏目标题。
- 不改七篇方法文章。
- 不改项目案例、公开作品或视觉系统。
- 不执行 GitHub push 或发布。
