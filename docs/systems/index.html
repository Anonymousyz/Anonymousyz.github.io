01 / 判断系统

如何确认真正的问题？

已有公开证据

先还原目标、事实和约束，再选择一个足以改变行动的工具。模型用于暴露盲区，不用于堆砌复杂感。

逻辑体系多元认知证据分级反方检查
怎样工作

面对一般问题，先回答“目标是什么、哪些是事实、下一步如何验证”。默认只调用一个主工具；新增模型已经不改变判断或行动时，立即停止。

公开证据

Research-to-Decision Toolkit 已把问题框定、证据矩阵、反方审查和决策包做成公开模板、CLI、合成案例与测试。

能力边界

思维框架不能证明结论正确。事实、引文和适用条件仍需回到可靠来源核验；人物视角只能提出问题，不能替代权威证据。

02 / 表达系统

如何让复杂内容被看懂？

正在公开化

写作负责组织判断，视觉负责降低认知成本。先确定这一页要让人理解什么，再处理语言、层级、颜色和留白。

写作方法结构审查视觉规范Design Tokens
怎样工作

写作从文种、对象和证据链开始；视觉遵循功能先于美学、一致性先于创造、删除先于添加，并保留一个清晰焦点。

公开证据

写作方法已经进入公开工具和案例页；视觉体系已有语义色板、间距与字号令牌、场景规范和机器可读 Tokens，正在整理为可公开复用的产物。

查看公开模板 ↗
能力边界

结构检查和机器规则可以发现偏差，但不能证明内容有判断力，也不能证明视觉“好看”。最终表达仍需要真实读者和人工审阅。

03 / 治理系统

如何让技术在边界内创造价值？

研究与应用

用 CIO 视角连接业务结果、企业能力、数据、AI、风险和技术投入，让治理成为可以执行和复查的门禁。

业务与 IT 对齐数据治理AI 治理风险与价值
怎样工作

每项技术投入都要说明业务结果假设；高风险 AI 用例需要风险分级、责任人、数据来源、评测记录和退出条件；成本与风险尽量使用业务语言表达。

公开证据

AI Prototype-to-Production Toolkit 已把部分治理要求转化为本地评分卡和否决项；CDMP 与 ITIL V4 构成数据管理和 IT 服务管理的基础训练。

能力边界

这是一套持续研究和应用的治理视角，不是 CIO 任职经历的替代品。框架覆盖面不等于真实组织已经完成落地。

04 / 构建系统

如何把方法变成可运行的工具？

公开实践

用软件开发和 SWE Agent 把方法写进模板、代码、测试和发布流程，让“已经完成”可以被命令、差异和版本记录验证。

软件开发SWE Agent测试与复现版本发布
怎样工作

从最小可验证任务开始：保留失败基线，做最小改动，运行测试，记录证据和权限边界。计划不能直接等于执行，发布也不能直接等于采用。

公开证据

目前已有三个公开仓库、正式 Release、可下载资产、测试或链接核验；SWE Agent 学习体系也以真实修复任务和教师侧 Harness 作为可信结论入口。

能力边界

公开代码证明产物存在并可检查，不证明已经形成广泛采用，也不等于成熟全栈工程履历。外部复现、Issue 和独立引用仍在积累。

这些系统的价值，不看页面或模型数量，而看它们能否持续产生更清楚的判断、更可靠的交付和可以复查的公开证据。
