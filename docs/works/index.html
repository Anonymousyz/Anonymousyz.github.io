四个公开仓库

四个仓库分别用来整理研究材料、检查 AI 原型投入真实业务前还缺什么、查找工程和治理资料，以及给方案和技术文档做署名前审稿。每个仓库都列出版本、样例、使用方法和限制。

01

材料很多，怎样把问题、证据和选择说清楚？

Research-to-Decision Toolkit

版本
v0.6.0
环境
Python 3.9+

把待决定的问题、证据来源、备选方案、尚不确定的事项和下一步行动整理成一份决策包。

示例测试Release决策包输出

使用限制工具只检查材料结构，不判断来源是否真实，也不代替负责人作决定。

三步试用

r2d init brief.json
r2d validate brief.json
r2d report brief.json --output decision_report.md

高质量 AI 写作

版本
v0.7.0
环境
Markdown 标准与 Agent Skills

按立意、逻辑、来源、分寸、字句、文气和得体程度检查文稿，并提供脱敏改稿示例和可安装的审稿技能。

评审标准脱敏判例测试Agent Skills

使用限制它只提供审稿标准和检查顺序，不能代替事实核查、专业审核，也不能替作者承担署名责任。

  1. 从 STANDARD.md 了解写前模式、一票否决与三层八维
  2. 按 principles/ 查看每个维度的问句、病征和改法
  3. 安装 pre-sign-review，在署名前做一次全检
