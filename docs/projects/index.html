工业绿色微电网评价软件

把分析师手里的资料、Excel 和报告流程做成可部署系统，覆盖数据导入、专业计算、审核、报告生成、权限和日志。

问题

分析师要在分散资料、Excel 和人工报告之间来回切换，团队需要一条可追溯、可复核的工作流。

我的工作

组织并参与需求拆解、工作流设计、架构决策、测试和部署验证。

怎样处理
  • 以分析师真实工作流确定产品边界，优先打通数据导入、计算、审核和报告生成。
  • 选择适合交付条件的部署方式，并保留权限、日志、测试和回退要求。
目前做到哪一步
  • 形成可部署版本及配套测试、部署和交付记录。
  • 把专业计算、项目数据和报告产物接入同一套流程。
交付路径

资料与 Excel → 数据导入 → 专业计算 → 审核 → 报告生成

  • 工作流结构
  • 系统范围
  • 测试与交付范围

02 / 从分析到决策 · 从技术到应用

工业技术改造投资管理平台项目

前期把政策和产业问题写成项目方案、任务和预算；建设期再把材料中的角色、对象和审批关系落实为台账、权限、状态和工作台。

问题

项目既要把政策目标变成可以申报和建设的任务，也要把建设材料中的角色、流程和监管要求放进系统。

我的工作

参与申报论证、预算与任务设计，也参与建设期的材料梳理、产品定义、流程与权限设计，以及实现检查和测试。

怎样处理
  • 申报阶段先把政策目标、建设任务、预算和验收要求对齐。
  • 建设阶段再梳理项目对象、角色、审批节点和退回路径，确定页面、权限与接口。
目前做到哪一步
  • 形成项目申报与论证材料、建设任务和预算安排。
  • 形成内部功能版本，覆盖项目台账、角色权限、审批状态和工作台；正式 UAT、预生产与生产验收尚未完成。

03 / 从技术到应用

口腔小程序项目

围绕家长、儿童和门诊协作，设计授权、检查报告、预约、随访与后台流程。

问题

家长、儿童和门诊需要在授权清楚的前提下查看报告、安排服务并完成后续协作。

我的工作

参与业务梳理、产品流程、系统实现检查和受控演示准备。

怎样处理
  • 把家长授权和儿童信息关系放在主要流程之前处理。
  • 用状态和权限限制报告、预约与后台操作。
目前做到哪一步
  • 形成小程序与管理后台的受控 Demo。
  • 完成主要流程和相关测试，保留试点前仍需处理的运行事项。
