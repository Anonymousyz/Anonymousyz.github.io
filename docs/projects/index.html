01 / 从分析到决策 · 从技术到应用

工业绿色微电网评价软件

这套系统沿用分析师处理资料、Excel 和报告的顺序，覆盖数据导入、专业计算、审核、报告生成、权限和日志。

问题

分析师原来要在分散的资料、Excel 和人工报告之间反复切换，团队需要一套便于追查数据来路、复核计算过程的工作流程。

我的工作

我组织需求梳理，参与工作流程设计、架构选择、测试和部署验证。

关键做法
  • 产品范围按分析师的实际工作顺序来定，先把数据导入、计算、审核和报告生成接起来。
  • 部署方式根据交付条件确定；权限、日志、测试和回退一并列入交付检查。
当前进度
  • 完成可部署版本，并留存测试、部署和交付记录。
  • 项目数据、专业计算和报告已经接入同一条工作流程。
工作路径

资料与 Excel → 数据导入 → 专业计算 → 审核 → 报告生成

02 / 从分析到决策 · 从技术到应用

工业技术改造投资管理平台项目

申报阶段先把政策要求写成项目方案、建设任务和预算；建设阶段再把材料中的项目、角色和审批关系写进台账、权限、状态和工作台。

问题

申报时，要把政策目标写成可以建设、可以验收的任务；建设时，还要把材料里的角色、流程和监管要求写进系统。

我的工作

我参与申报论证、预算和建设任务设计。进入建设阶段后，继续梳理材料，并参与产品定义、流程与权限设计、功能检查和测试。

关键做法
  • 申报阶段逐项核对政策目标、建设任务、预算和验收要求。
  • 建设阶段梳理项目对象、角色、审批节点和退回路径，再据此确定页面、权限和接口。
当前进度
  • 完成项目申报与论证材料，明确建设任务和预算安排。
  • 已完成内部功能版本，覆盖项目台账、角色权限、审批状态和工作台。正式用户验收测试、预生产验证和生产验收尚未完成。

03 / 从技术到应用

口腔小程序项目

围绕家长、儿童和门诊的协作，先梳理授权关系，再把检查报告、预约、随访和后台处理接起来。

问题

家长明确授权范围并关联儿童信息后，才能查看检查报告、预约服务并继续随访。

我的工作

我参与业务梳理和产品流程设计，也检查系统实现，并准备受控演示。

关键做法
  • 先确定家长授权与儿童信息的关系，再进入报告和预约。
  • 用状态和权限分别控制报告查看、预约和后台操作。
当前进度
  • 已完成小程序和管理后台的受控演示版。
  • 主要流程和相关测试已经完成；试点前仍有运行事项需要处理。
