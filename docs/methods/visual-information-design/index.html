从技术到应用 · 05

视觉功能 · 一致性 · 删除

视觉规则帮助读者找到重点、理解关系、辨认状态并完成动作。设计先处理功能，再建立一致性，并通过删除减少无关负担；删除以后仍要看得出重点和层级。

方法路径

三个原则

  1. 01功能先于美学
  2. 02一致性先于创造
  3. 03删除先于添加

功能先于美学

先确定读者要找什么、比较什么、判断什么或完成什么动作，再安排焦点、层级和阅读顺序。页面任务没有说清时，美化只会把结构问题包进更精细的样式。

视觉选择要能说明作用：字号区分层级，留白组织信息组，颜色标识含义和状态，对齐帮助比较。无法说明作用的元素，需要重新检查。

一致性先于创造

相同的信息层级、组件和状态使用相同表达，读者便不必反复学习。新表达应服务新的任务，不能只为制造差异。

不同载体可以有不同外观。Word、PPT、数据表格、网页和软件界面的阅读方式各不相同，但标题、颜色、数字、状态和图表应保持共同语义。

删除先于添加

页面显得平或重点不清时，先检查重复标题、解释性口号、无效装饰和层级竞争。删掉这些内容，主任务与关键信息才有空间。

删除时保留必要的上下文、操作反馈和可访问性说明。简洁不能省掉条件，也不能让读者猜测状态。

在视觉的甜蜜区里取舍

信息不能太少，也不能被装饰淹没。读者要看得出重点、关系和状态，也不该被无关形式打断。先让任务完成，再用一致的规则降低理解成本，删掉不服务任务的形式。

删减以后如果层级消失、状态难辨或操作缺少反馈，就应恢复必要信息。克制是一种取舍，不是把所有页面都压成同一种空白版式。

用 Design Tokens 固定共同语言

颜色、字体、字号、间距、圆角和边框以语义化 Design Tokens 记录，例如正文颜色、危险状态和章节间距，而非只保存某个数值。

跨载体使用时先保留语义，再根据介质调整尺寸与排版。这样可以追踪设计选择，也能减少设计稿、文档和代码之间的漂移。

机器可读规则仍在形成

现在我能说明设计语义和人工检查方向。机器可读的 Token、组件状态约束、文档样式映射和自动检查规则还在补；公开页面不会把它们写成已经具备的能力。

每增加一项机器规则，都要对应一个反复出现的真实问题，并提供检查结果与人工复核入口。自动化先找出偏差，具体场景仍由人判断。

为不同载体安排不同读法

Word 支持长文查阅，需要稳定标题、页码、图表题注和重复表头；PPT 每页服务一个主要判断；数据表格要明确字段、单位、缺失值和比较方向。

网页强调导航、链接名称和响应式阅读；软件界面还要处理输入、确认、加载、空状态、错误与恢复。移动端优先保留任务和关键内容。

分级检查，并记录有意偏离

快速检查用于发现焦点、层级、阅读顺序、对比度和明显状态缺失；完整诊断再覆盖组件一致性、不同尺寸、键盘操作、表单标签、图像说明和跨载体表达。

有理由偏离设计规则时，记录原因、影响范围和回收条件。为具体任务偏离规则可以，但没有记录会逐步破坏共同语义。

使用说明

适用边界

  • 视觉规则需要结合任务、内容和受众验证，不能只凭设计者偏好判断。
  • 固定尺寸和阈值受载体、设备和内容影响，不作为跨场景的普遍结论。
  • 机器可读规则正在形成；公开页面只说明当前已经采用的原则与检查方式。
  • 可访问性属于功能质量，需要与视觉检查一起完成。
