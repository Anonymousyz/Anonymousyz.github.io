Research-to-Decision Toolkit · v0.6.0

研究材料怎样进入
一个决策包

这是一个本地运行的公开工具和模板包，用来把问题、证据、分歧、选项和下一步整理成一份决策包。

查看 v0.6.0 Release ↗打开 GitHub 仓库 ↗

01 / 它解决什么

材料很多，决定仍然卡住

项目里积累了很多资料后，负责人仍可能说不清：现在要决定什么、有哪些现实选项、哪些证据足以支持建议、谁对下一步负责。

常见结果

一份看起来完整的材料

有背景、数据和结论，但决策人、备选项、反证、退出条件和后续责任仍然模糊。

R2D 的目标

一份能经受讨论的决策包

把证据和判断分开，让负责人能追问、反对、补证据或明确暂不决策。

02 / 怎样工作

四步，保留人的判断

工具固定材料结构。来源核验、领域判断、风险承担和最终决定，由具体责任人完成。

01

把题目变成决策

明确谁需要决定什么、截止时间是什么、如果不决定会怎样。

02

把材料变成证据

区分事实、判断、假设和待核验项；记录最薄弱的一环。

03

把结论变成选项

列出备选方案、取舍、风险条件，以及什么证据会改变建议。

04

留下可复查的产物

记录决策包、责任人和反馈，让下一次讨论能回到原来的判断。

03 / 能直接检查什么

可以运行和检查的公开产物

以下内容可在公开仓库中检查或复现。案例数据为虚构或合成数据，与真实组织、部署和结果无关。

模板问题框定、证据矩阵、决策备忘录、反馈记录↗合成案例完整的 AI 治理决策简报、报告和未填反馈日志↗CLI 与测试初始化、校验、评分与报告命令；结构与否决项测试↗方法状态评分规则、已完成测试和待校准项↗

04 / 60 秒试用

先运行一个虚构样例，
再换成获授权的材料。

python -m pip install "https://github.com/Anonymousyz/research-to-decision-toolkit/releases/download/v0.6.0/research_to_decision_toolkit-0.6.0-py3-none-any.whl"
r2d init brief.json
r2d validate brief.json
r2d report brief.json --output decision_report.md

05 / 可信度边界

目前做到哪一步

我把结构检查、来源核验和实际采用分开记录。当前版本完成结构和命令测试；外部复现和实际采用的记录还在积累。

已验证

工具、样例和版本可复查

  • 公开仓库、Release、MIT 许可证、模板和合成案例均可访问。
  • CLI 可以校验结构、评分规则和否决项，并输出决策报告。
  • 方法状态文档明确列出测试、来源声明与人工复核的边界。
尚未验证

仍需领域判断和外部验证

  • 24 分评分是作者设计的结构化启发式，不代表决策获批、正确或可实施。
  • CLI 不会自动抓取、认证或判断来源是否足以支持某一主张。
  • 外部复现、独立评审与真实采用证据仍在积累。
