Research-to-Decision Toolkit · v0.6.0

怎样把研究材料整理成
一个决策包

这套公开工具在本地运行，可以把待决定的问题、证据、分歧、几种选择和下一步行动整理成一份决策包。

查看 v0.6.0 Release ↗打开 GitHub 仓库 ↗

01 / 它解决什么

材料很多，仍然作不了决定

项目积累了不少资料，负责人仍可能说不清现在要决定什么、有哪些可选做法、哪些证据足以支持建议，也不知道下一步由谁负责。

常见结果

一份内容齐全的材料

材料有背景、数据和结论，却没有写清谁来决定、还有哪些选择、什么证据会推翻结论、何时退出，以及后续由谁负责。

R2D 的目标

一份便于讨论和追问的决策包

证据和判断分开记录，负责人可以追问、提出不同意见、补充证据，也可以明确暂缓决定。

02 / 怎样工作

四步整理，不替人作决定

工具只负责整理材料结构。来源是否可靠、专业判断是否成立、风险能否承担、最终作什么决定，都由具体负责人处理。

01

先说清要作的决定

写清谁要决定什么、何时决定，以及继续拖延会有什么代价。

02

核对材料能否支持判断

分开事实、判断、假设和待核对事项，并标出证据最薄弱的地方。

03

把可选做法摆在一起比较

列出几种做法、各自取舍和风险，也说明出现什么证据就要修改建议。

04

留下可以复查的记录

保存决策包、负责人和实际反馈，下次讨论可以据此复核原判断。

可以直接查看和运行的内容

以下内容都能在公开仓库中查看或复现。案例使用虚构或合成数据，不对应任何真实组织、部署环境和项目结果。

模板问题框定、证据矩阵、决策备忘录、反馈记录↗合成案例一份完整的 AI 治理决策简报、报告和空白反馈记录↗CLI 与测试初始化、校验、评分与报告命令；结构与否决项测试↗方法状态评分规则、已经完成的测试和尚待校准的内容↗

04 / 60 秒试用

先运行虚构样例。
使用真实材料前，请确认已经获准处理。

python -m pip install "https://github.com/Anonymousyz/research-to-decision-toolkit/releases/download/v0.6.0/research_to_decision_toolkit-0.6.0-py3-none-any.whl"
r2d init brief.json
r2d validate brief.json
r2d report brief.json --output decision_report.md

05 / 当前进展

目前做到哪一步

当前只验证工具能否运行、结构规则是否生效。来源是否可靠、专业判断是否成立和材料是否被采用，另行记录；外部复现和实际使用记录仍在积累。

已验证

工具、样例和版本均可查看

  • 公开仓库、发布版本、MIT 许可证、模板和合成案例均可访问。
  • CLI 可以校验结构、评分规则和否决项，并输出决策报告。
  • 方法状态文档列出了已经完成的测试、来源说明和需要人工复核的事项。
尚未验证

仍要经过专业判断和外部验证

  • 24 分评分只是一套材料结构检查方法，不能据此认定决策已经获批、结论正确或方案可以实施。
  • 命令行工具不会自动抓取或认证来源，也不会判断材料是否足以支持某项结论。
  • 外部复现、独立评审和实际使用记录仍在积累。

下一步需要用一份已经获准处理的材料走完流程，并如实记录遇到的问题。
