# Editorial Portfolio Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current evidence-dashboard presentation with a Chinese editorial portfolio that leads with “从分析到决策 / 从技术到应用”, adds four complete method pages, and preserves truthful public project boundaries.

**Architecture:** Keep Astro's static-site structure and centralize public copy in data modules. Build the homepage from three small index components, generate four static method routes from one article component, and apply a shared editorial stylesheet after the existing stylesheet so the migration can proceed without breaking old routes midway. Keep project, works, about, legacy, and tool-case routes live throughout.

**Tech Stack:** Astro 7, JavaScript ES modules, CSS, Node.js built-in test runner, static GitHub Pages output in `docs/`.

**Source Spec:** `planning/superpowers/specs/2026-07-19-editorial-portfolio-redesign-design.md`.

**Implementation status (2026-07-19):** Tasks 1–6 are complete and verified. The long microgrid case and the final media/content for the two Demo presentations remain deliberately deferred.
